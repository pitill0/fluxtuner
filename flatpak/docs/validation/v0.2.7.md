## Summary

FluxTuner `v0.2.7` adds a History view to the GTK GUI and aligns GUI playback history behavior with the existing TUI history flow.

This release also fixes the AppStream `<releases>` metadata structure so the Flatpak metainfo validates correctly.

---

## Changes validated

### GTK GUI History

Validated change:

- Added a **History** view to the GTK GUI.
- GUI playback now writes to the shared playback history.
- GUI can display stations previously played from the GUI.
- GUI can display stations previously played from the TUI.
- Stations can be played from History.
- Stations can be added to favorites from History.
- `▶` and `★` markers remain coherent in History.

### AppStream / Flatpak metadata

Validated change:

- Added release metadata for `0.2.7`.
- Corrected the AppStream `<releases>` structure.
- Confirmed AppStream validation succeeds.
- Known pedantic warning remains accepted:
  - `cid-contains-uppercase-letter io.github.pitill0.Fluxtuner`

Decision:

- Do not rename the app-id only to silence the pedantic warning.
- Keep `io.github.pitill0.Fluxtuner` consistent across manifest, desktop files, metainfo and icon names.
