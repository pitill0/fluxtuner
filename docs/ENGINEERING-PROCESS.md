# Engineering Process

FluxTuner is developed using an AI-assisted workflow under human direction.

Artificial intelligence tools may assist with implementation, refactoring, testing, documentation, and analysis, but all project decisions remain under human control and responsibility.

## Principles

* Human-defined product vision and roadmap.
* Human-defined requirements and scope.
* Human-led architectural decisions.
* AI-assisted implementation under human direction.
* Human review before acceptance.
* Human validation before release.
* Human responsibility for maintenance and support.

## Responsibility Model

| Area                       | FluxTuner approach                |
| -------------------------- | --------------------------------- |
| Product direction          | Human-defined                     |
| Requirements and scope     | Human-defined                     |
| Architecture               | Human-led                         |
| Implementation             | AI-assisted under human direction |
| Refactoring                | AI-assisted and human-reviewed    |
| Automated testing          | Tool-assisted and human-reviewed  |
| Manual validation          | Human-performed                   |
| Pull request acceptance    | Human decision                    |
| Release decision           | Human decision                    |
| Maintenance responsibility | Human-owned                       |

## Development Workflow

A typical contribution follows this process:

1. Define the objective or problem.
2. Design or discuss the proposed solution.
3. Implement changes.
4. Run automated checks and tests.
5. Perform manual validation.
6. Open a pull request.
7. Review the changes.
8. Merge only after all required checks pass.
9. Publish a release when the change is considered production ready.

For detailed contributor guidelines, see [CONTRIBUTING.md](../CONTRIBUTING.md).

## AI Usage

AI tools may be used to:

* Generate implementation drafts.
* Suggest refactorings.
* Assist with documentation.
* Help design tests.
* Analyze code and project structure.
* Explore alternative approaches.

AI output is not considered accepted code simply because it was generated successfully.

All changes are expected to pass the project's validation and review process before becoming part of an official release.

## What This Means

FluxTuner embraces modern development tools while maintaining human accountability.

The use of AI does not replace engineering judgment, review, testing, or release responsibility.

Project ownership, architectural direction, and release decisions remain human responsibilities.
